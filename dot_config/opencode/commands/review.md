---
description: Ревьюит открытый PR через @code-reviewer и публикует комментарий
---

Текущая ветка: !`git branch --show-current`

Шаги:

1. **Проверь окружение:**
    - Проверь, доступен ли `gh` CLI (`gh --version`). Если недоступен — сообщи об ошибке и прекрати выполнение

2. **Определи PR:**
    - Если передан аргумент и он не пуст (`$ARGUMENTS` определён и не является пустой строкой):
        - Проверь, что это либо число (только цифры), либо URL вида `https://github.com/*/pull/\d+`
        - Если это число — используй его как номер PR
        - Если это URL — извлеки номер PR из URL
        - Если аргумент не соответствует ни одному формату — сообщи пользователю об ошибке: "Некорректный аргумент. Укажите номер PR (число) или ссылку на PR вида https://github.com/owner/repo/pull/123" и прекрати выполнение
    - Иначе получи PR для текущей ветки: `gh pr view --json number,url,headRefName,baseRefName,title`
    - Если команда завершилась ошибкой или PR не найден — сообщи пользователю и прекрати выполнение

3. **Получи изменения:**
    - Выполни `gh pr diff <number>` и сохрани результат в переменную
    - Если команда завершилась ошибкой — сообщи пользователю текст ошибки (выведи stderr) и прекрати выполнение
    - Если diff пуст — сообщи пользователю что в PR нет изменений и прекрати выполнение
    - Если количество строк diff превышает 2000 — предупреди пользователя: "Diff содержит N строк. Это может превысить контекстное окно LLM." и спроси, хочет ли он продолжить

4. **Запусти ревью:**
    - Передай результат агенту `@code-reviewer` для анализа
    - Дождись результата

5. **Проверь результат ревью:**
    - Убедись, что результат содержит ожидаемую структуру (наличие эмодзи критичности: 🔴, 🟠, 🟡, ⚪)
    - Если результат не содержит ни одного из этих маркеров — скорее всего, @code-reviewer вернул ошибку
    - При невалидном результате — сообщи пользователю текст ошибки и не публикуй комментарий

6. **Опубликуй результат:**
    - Передай тело комментария через heredoc, чтобы избежать проблем с shell-интерпретацией специальных символов:
        ```
        gh pr comment <number> --body-file - << 'EOF'
        <результат ревью>
        EOF
        ```
    - Если команда завершилась ошибкой — сообщи пользователю текст ошибки и предложи опубликовать комментарий вручную
    - Сообщи пользователю ссылку на PR и что ревью опубликовано
