---
description: Удаляет локальные и удалённые ветки, влитые в main/master (включая squash merge и rebase merge)
agent: build
---

Определи базовую ветку: сначала проверь `main` через `git show-ref --verify --quiet refs/heads/main`, если нет — используй `master`.

Если текущая ветка (`git branch --show-current`) НЕ является базовой — выполни `git checkout <base>`.

Выполни `git pull --ff-only` (базовая ветка должна быть актуальна для корректного определения влитых веток).

Выполни `git fetch origin --prune`.

Собери список веток на удаление в два этапа:

**Этап 1: Ветки, влитые через merge commit или fast-forward.**
Выполни `git branch --merged <base>`. Исключи из списка `main`, `master`, `HEAD` и строки, начинающиеся с `*`. Добавь оставшиеся ветки в список на удаление.

**Этап 2: Ветки, влитые через squash merge или rebase merge.**
Для каждой ветки, которая не попала в Этап 1 и не является `main`/`master`/`HEAD`/`*`:
1. Проверь `git diff <base> <branch>` с флагом `--exit-code --quiet`. Если код возврата 0 — diff пустой, все изменения ветки уже влиты в базовую ветку (squash/rebase merge). Добавь такую ветку в список на удаление.
2. Если diff не пустой — пропусти ветку (она содержит незалитые изменения).

Если список веток на удаление пуст — сообщи пользователю, что нет влитых веток, и заверши выполнение.

Иначе для каждой ветки из списка:
1. Удали на удалённом репозитории: `git push origin --delete <branch>`
2. Удали локально: `git branch -d <branch>`

Сообщи пользователю результат: сколько веток удалено и какие именно. Отдельно укажи, какие ветки найдены через --merged, а какие — через проверку diff.

Важно: никогда не трогай `main`/`master`. Всегда переключайся на базовую ветку перед удалением.
