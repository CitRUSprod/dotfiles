---
description: Создаёт Pull Request из текущей ветки в main/master через gh CLI.
---

Важно: команда не создаёт коммиты. Никогда не выполняй `git commit`.

Сначала изучи стиль описаний PR в проекте:

```
gh pr list --state merged --limit 10 --json title,body
```

Затем выполни шаги:

1. Определи текущую ветку (`git branch --show-current`).

2. Определи базовую ветку: проверь существование `main` (`git show-ref --verify --quiet refs/heads/main`), если нет — используй `master`.

3. Если текущая ветка совпадает с базовой или является master/main — сообщи, что нужно сначала переключиться на фича-ветку и прекрати выполнение.

4. Проверь, нет ли уже открытого PR для текущей ветки: `gh pr view --json number,url 2>/dev/null || true`. Если открытый PR есть — сообщи номер и ссылку на существующий PR, затем прекрати выполнение.

5. Найди точку расхождения: `git merge-base <base> HEAD`.

6. Собери список коммитов: `git log <merge-base>..HEAD --oneline --format="%s"`.

7. Если коммитов нет — сообщи, что нет коммитов для PR, и прекрати выполнение.

8. Сгенерируй заголовок и тело PR на основе анализа коммитов (`git log <merge-base>..HEAD`). Если пользователь передал аргументы (`$ARGUMENTS`), учти их как дополнительные указания при генерации заголовка и тела (но не используй как готовый заголовок). Язык заголовка и тела — тот же, что и в проекте.

9. Если сессия только для чтения (plan mode), просто выведи предложенные заголовок и тело PR без внесения изменений.

10. Если текущая сессия позволяет выполнять команды (build mode), сначала проверь, доступен ли `gh` CLI (`gh --version`). Если недоступен — сообщи об ошибке. Иначе выполни:

```
gh pr create --base <base> --head <current> --title "<title>" --body "<body>"
```

Сообщи пользователю ссылку на созданный PR (вывод команды).
